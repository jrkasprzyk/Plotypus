# Refactor Plan: Split `plotypus/problems.py` into a `problems/` package by family

## Goal & Scope
**Maintainability** for a teaching / personal codebase. Plotypus has **no users yet**, so
this refactor owes **no backward-compatibility ceremony** (no cross-version state-file
compat, no lazy-import optimization). The `__init__` re-exports eagerly. The split exists
to make families navigable and to mirror code structure in the docs.

## Current State
- Single file `plotypus/problems.py` (1629 lines, 50 classes) holds **all** benchmark problems + module-private helpers.
- Families: DTLZ (1–4,7), WFG (base + 1–9 + ~40 `_`-helpers), UF (1–13), CF (1–10), ZDT (base + 1–6), Simple (Schaffer, Belegundu), Combinatorial (Knapsack, TSP, PortfolioOptimization).
- Shared private helpers:
  - WFG transition/shape/reduction funcs (`_normalize_z`, `_s_linear`, `_WFG1_t1..t4`, `_WFG*_shape`, `_r_nonsep`, `_calculate_f`, …) → used by WFG **and** `UF13`.
  - `_transform` → used by `UF11`, `UF12` only (stays in `uf.py`).
- Cross-family class references (verified): `UF11 → DTLZ2` (line 1028), `UF12 → DTLZ3` (line 1082), `UF13(WFG)`. Nothing else crosses families.
- Original imports (verified): `functools, math, operator, random`, `from abc import ABCMeta`, `from ._math import EPSILON`, `from .core import Constraint, Direction, Problem, Solution`, `from .types import Binary, Permutation, Real`.
- Consumers of the public names:
  - `plotypus/__init__.py` (lines 62–67) re-exports the full list from `.problems`.
  - `gui/registries.py` imports concrete classes from top-level `plotypus`.
  - `plotypus/tests/test_problems.py` imports `WFG, ZDT` and **introspects the `plotypus.problems` namespace** (`inspect.getmembers`) to auto-discover Problem subclasses.
  - `examples/` import concrete classes from top-level `plotypus`.
  - `docs/api/plotypus.problems.rst` → machine-generated by `sphinx-apidoc` (command recorded in `docs/conf.py:41`).

## Target State
`plotypus/problems.py` becomes a package. **External contract unchanged**: `plotypus.problems.<Class>` and `from plotypus import <Class>` resolve identically. (Note: class `__module__` moves to the submodule, e.g. `plotypus.problems.dtlz` — accepted, see Goal & Scope.)

```
plotypus/problems/
  __init__.py        # re-exports ALL public classes (+ WFG, ZDT bases) — keeps namespace stable
  _wfg_functions.py  # pure WFG transition/shape/reduction funcs (depends only on _math.EPSILON)
  dtlz.py            # DTLZ1, DTLZ2, DTLZ3, DTLZ4, DTLZ7
  wfg.py             # WFG base + WFG1..WFG9   (imports _wfg_functions)
  zdt.py             # ZDT base + ZDT1..ZDT6
  uf.py              # UF1..UF13 + _transform  (imports dtlz, wfg, _wfg_functions)
  cf.py              # CF1..CF10               (self-contained)
  simple.py          # Schaffer, Belegundu
  combinatorial.py   # Knapsack, TSP, PortfolioOptimization
```

Per-**family** chosen over per-**problem**: per-problem = 40+ tiny files + heavy shared-helper plumbing for no gain. User listed families first.

`_wfg_functions.py` (not `_wfg_base.py`): the file holds transition funcs, shape funcs (`_convex`, `_concave`, `_mixed`, `_disc`), reduction (`_r_sum`, `_r_nonsep`) and `_calculate_f` — not a base *class*. It is the strongest pedagogical unit: a self-contained mathematical vocabulary the WFG problems compose.

### Dependency graph (target, acyclic — verified)
```
_wfg_functions  <- wfg <- uf
dtlz            <- uf
(core, types, _math) <- relevant modules
__init__ <- all
```
`uf` is the only multi-dep module; nothing imports `uf` back → no circular import.

## Risks (post-trim)
- **Circular import** (`uf` ↔ `wfg`/`dtlz`): graph is strictly acyclic and verified — mitigated.
- **Silent test-coverage drop**: if `__init__.py` forgets a class, introspection finds fewer problems and the suite still "passes" with fewer cases. Mitigated by the Phase 0/2 discovered-set diff (must be empty).
- **CI lint failure**: unused per-module imports → F401. Mitigated by the Phase 3 `flake8` step.
- **Verbatim-copy errors** (dropped helper, wrong import): mitigated by the full test suite + discovered-set diff.

## Decisions resolved during planning (grill-me)
- Goal = maintainability; no users → no back-compat/serialization concern; eager re-export.
- `__module__` moves to submodules — accepted (no cross-version state-file compat owed).
- Shared helpers kept in one file, named `_wfg_functions.py`.
- No `git mv` blame-preservation (accept blame reset; simpler).
- Cut wheel-inspection + concatenation byte-diff; trim rollback to one line.
- Docs via apidoc re-run (not hand-edited `:imported-members:`); include `_wfg_functions` page.
- Two commits (code, then docs).
